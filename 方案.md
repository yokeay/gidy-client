# gidy-client WS + ECH 穿透 GFW

## 一、客户端 DATA 帧格式（已实现）

### 上行（客户端→服务端）：`[0x06][实际数据]`
- 帧类型 1 字节 + 原始数据，无 padding，无长度前缀

### 下行（服务端→客户端）：`[0x06][2字节长度BE][数据][padding]`
- 2 字节大端长度 = 实际数据字节数
- padding 紧跟数据之后，客户端按长度字段剥离

```rust
// 客户端发送（TCP→WS）
let mut frame = Vec::with_capacity(1 + n);
frame.push(0x06);                    // FRAME_DATA
frame.extend_from_slice(&buf[..n]);  // 实际数据，无 padding

// 客户端接收（WS→TCP）
let actual_len = u16::from_be_bytes([data[1], data[2]]) as usize;
let actual_data = &data[3..3 + actual_len];
tcp_write.write_all(actual_data).await;
```

## 二、完整帧协议

| 帧类型 | 值 | 方向 | 格式 |
|--------|-----|------|------|
| AUTH_REQUEST | 0x01 | C→S | `[0x01][auth_string]` |
| AUTH_RESPONSE | 0x02 | S→C | `[0x02][status: 0x00=OK]` |
| CONNECT | 0x03 | C→S | `[0x03][authority]` |
| CONNECT_OK | 0x04 | S→C | `[0x04]` |
| CONNECT_FAIL | 0x05 | S→C | `[0x05][status]` |
| DATA (上行) | 0x06 | C→S | `[0x06][data]` |
| DATA (下行) | 0x06 | S→C | `[0x06][2B len BE][data][padding]` |
| CLOSE | 0x07 | 双向 | `[0x07]` |
